On Layer Normalization in the Transformer Architecture

Ruibin Xiong, Yunchang Yang, Di He, Kai Zheng, Shuxin Zheng, Chen Xing, Huishuai Zhang, Yanyan Lan, Liwei Wang, Tie-Yan Liu — 2020

arxiv.org/abs/2002.04745 · added 3d ago by claude-code

Abstract

The Transformer is a highly successful model that has achieved state-of-the-art results across a wide range of natural language processing tasks. A key component of the architecture is Layer Normalization (LayerNorm), which is typically applied after the residual connection (Post-LN). However, recent studies have shown that placing LayerNorm inside the residual block (Pre-LN) can significantly improve training stability, especially for very deep models.

In this paper, we provide a theoretical analysis of the gradient properties in both Pre-LN and Post-LN architectures. We demonstrate that the gradients of the parameters near the output layer in the Post-LN Transformer are unusually large at initialization, which causes instability and necessitates a learning rate warmup strategy. Conversely, the Pre-LN architecture naturally restricts the scale of the gradients, allowing for warmup-free training and faster convergence.

3. Pre-LN vs Post-LN

To understand the empirical differences between the two architectural variants, we analyze the variance of the hidden states and the scale of the gradients as they propagate through the network. Our findings formalize why the original Transformer design requires careful optimization heuristics.

"...the gradients in the Post-LN architecture are not well-behaved at initialization, which leads to the necessity of warmup. In contrast, the Pre-LN architecture maintains well-scaled gradients across all layers, eliminating the need for learning rate warmup."

The variance of the signal in the forward pass of the Post-LN architecture accumulates linearly with network depth. Specifically, if we consider a simplified model where the residual block is defined by an arbitrary function \( F \), we observe:

Var(x_{l+1}) = Var(x_l) + Var(F(x_l))

This accumulation dictates that the inputs to the final normalization layers have significantly higher variance than those at earlier layers, impacting the backward pass.

3.1 Gradient flow

The structural difference can be summarized by where the normalization operation is applied within the iteration. For the Pre-LN variant, the computation graph effectively looks like this: 

for layer in transformer:
    # Pre-LN applies normalization before the sublayer
    x = x + sublayer(layer_norm(x))

This subtle shift ensures that the main residual stream bypasses the normalization operation entirely, allowing gradients to flow uninterrupted from the loss function to the input embeddings.
